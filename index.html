HTML Form Reference

Form Tags

Tag Name Description
<form> Form

Collects and sends information to an external source. It can contain multiple inputs and be submitted.

<input> Input

Renders a variety of input fields on a webpage. The way it's rendered is determined by the type attribute.

<label> Label

Provides identification for a specific <input> based on matching values with the <input>'s id attribute and the <label>'s for attribute. By default, clicking on the <label> will focus the field on the related <input>.

<textarea> Text Area

Creates a text-box for multi-line input. When rendered by the browser, it can be stretched/shrunk by the user, even tho you can determine its initial size, with the rows and cols attributes.

<select> Select

Creates a dropdown list. The list of options of the dropdown are created with the <option> tag. By default only one <option> can be selected at a time.

<datalist> Data list

Achieves a basic search/autocomplete functionality by pairing itself with an <input>. To do it, the <option>'s list attribute's value must match the value of <datalist>'s id. The list items of this tag are specified as <option> elements nested inside it.

Form Input Attributes

Attribute Value Description
type "text"

Renders a single row input field that users can type text inside.

type "number"

Allows the user to enter only numbers and a few special characters inside the field.

type "checkbox"

Renders a single checkbox item. To create a group of checkboxes related to the same topic, they should all use the same name attribute. These checkboxes may have their own value attributes, to represent the name of the checkbox selected between the <input> tags.

type "range"

Creates a slider that will act as a selector between a minimum and a maximum value. These values are set using the min and max attributes respectively. The slider can be adjusted to move in different steps or increments using the step attribute.

type "radio"

Renders a single radio button. Multiple radio buttons of a related topic are given the same name attribute value. Only a single option can be chosen from a group of radio buttons.

type "password"

Renders a single row input field which allows the user to type censored text inside the field. It is used to type in sensitive information.

type "submit"

A submit button is rendered. It submits, by default, the <form> and executes its action.

required none

Specifies that the field must include a value.

min "a number"

Specifies the minimum value that can be entered into the field in input tags with type number.

max "a number"

Specifies the maximum value for an <input> field with the type="number".

name "a name"

In order for a form to send (or submit) data, it needs to be able to put it into key-value pairs. So the name attribute becomes the key and the value of the input becomes the value. That value will only be submitted if the name attribute is specified.

minlength "a number"

Supports minimum length validation to an input field with type text.

maxlength "a number"

Specifies the maximum number of characters that can be entered into a field of an input with the type="text".

pattern "a regex"

Uses a regular expression to match against (or validate) the value of the <input> with type="text".

HTML Form Validator

HTML forms allow you to specify different kinds of validation for your input fields to make sure that data is entered correctly before being submitted. HTML supports a number of different validators, including things like minimum value, minimum/maximum length, etc. The validators are specified as attributes on the <input>field.

Submitting a Form

Once we have collected information in a form we can send that information somewhere else by using the action and method attributes. The action attribute tells the form to send the information. A URL is assigned that determines the recipient of the information. The method attribute tells the form what to do with that information once it's sent. An HTTP verb is assigned to the method attribute that determines the action to be performed.
